先學會操作 Codex：我的網頁設計第一步

學會把想法變成清楚的指令，再一步一步完成自己的網站。

01DESIGNING THE WEB

開始學習網頁設計時，我原本以為最重要的是先學會很多程式語法，例如 HTML、CSS、JavaScript。後來我發現，除了自己學會基本觀念之外，懂得如何使用工具也很重要。Codex 就是我目前學習網站製作時很重要的助手。

Codex 可以幫助我檢查網站檔案、修改程式碼、找出錯誤，也可以依照我的需求提出改善建議。對初學者來說，這不代表可以完全不用學習，而是可以讓我更快知道問題在哪裡，也更清楚每一次修改的原因。

第一個重點：把需求說清楚

我覺得操作 Codex 的第一個重點，是要把需求說清楚。例如不要只說「幫我改網站」，而是要說明我要改哪個頁面、想要達成什麼效果、不要動到哪些內容。像是：「請幫我改善首頁，加入最新文章區、作品展示區，並保持原本風格。」這樣 Codex 比較能正確理解我的目標。

第二個重點：一次完成一件事

第二個重點，是不要一次要求太多。剛開始可以先請 Codex 檢查首頁，再請它修改一個區塊，確認沒問題後再繼續下一步。這樣比較容易掌握網站的變化，也比較不會因為一次改太多而看不懂。

第三個重點：完成後一定要檢查

第三個重點，是完成後一定要檢查結果。Codex 可以協助我寫程式，但網站最後呈現給使用者看的效果，還是需要我自己確認。例如文字是否清楚、手機版是否好閱讀、按鈕是否明顯、文章區有沒有正常顯示。

這次建立「設計網頁之路」網站，就是我學習操作 Codex 的開始。我希望透過每一次練習，把不懂的地方記錄下來，也把修改網站的過程變成自己的學習筆記。未來我會繼續用 Codex 協助我改進網站，同時也慢慢學會 HTML、CSS 和網頁設計的基礎。

學會操作 Codex，不只是學會使用一個工具，而是學會如何把想法變成清楚的指令，再一步一步完成自己的網站。
